# AGENTS.md - Spec-Kit Configured Project

## Project Overview
A skeuomorphic-style Todo List application built using Spec-Driven Development workflow.

---

## Spec-Kit Commands (Available in OpenCode)

When working on this project, use these slash commands to guide development:

### Core Workflow

**1. Define Requirements**
```
/speckit.specify [your requirements here]
```
Use this to describe what features you want to build. Focus on the **what** and **why**, not the tech stack.

**2. Create Technical Plan**
```
/speckit.plan
```
After specifying requirements, use this to create a technical implementation plan with your chosen tech stack.

**3. Generate Tasks**
```
/speckit.tasks
```
Break down the plan into actionable tasks.

**4. Execute Implementation**
```
/speckit.implement
```
Execute all tasks to build the feature according to the plan.

**5. Update Project Principles**
```
/speckit.constitution
```
Create or update governing principles for code quality, testing standards, UI consistency, and performance.

## Development Principles

**Language Rules:**
- User communicates in **Chinese**
- AI assistant communicates in **English**
- All code comments should be in English

**Tech Stack:**
- Vanilla HTML/CSS/JavaScript (no frameworks)
- Local storage for data persistence
- Skeuomorphic design aesthetics

**Design Guidelines:**
- Realistic textures and shadows
- Physical metaphors (buttons that press in, lists that feel tactile)
- Smooth animations and transitions
- Warm color palette inspired by physical objects

**Testing Standards:**
- Manual testing via browser
- No automated test framework required
- Validate functionality across Chrome/Firefox
