# /speckit.constitution - Project Constitution Generator

When the user provides a description of what they want to build, generate a comprehensive constitution document that establishes:

1. Technical standards and stack choices
2. Code quality and naming conventions  
3. Design principles (including skeuomorphic aesthetics if relevant)
4. Testing protocols
5. Version control guidelines
6. Configuration management
7. Amendment process

Read the skeleton.constitution file as a template, then customize based on user requirements.

**Output**: A complete MDX/MD document in the project root named `skeleton.constitution` or update existing one.
